Fundamentos Iniciais

A Arte da Programação

Um programa de computador é um conjunto de instruções que orienta o hardware sobre o que deve ser feito. Softwares resolvem problemas do mundo real e são escritos em linguagens de programação.

O que é Python?

Python é uma linguagem de alto nível criada no início dos anos 1990 pelo holandês Guido van Rossum. É uma das linguagens mais populares do mundo e tem crescimento contínuo em áreas como machine learning, inteligência artificial e ciência de dados.

Em 2020, a IEEE Computer Society a classificou como a linguagem mais indicada para aprender devido à sua eficiência e versatilidade.

Arquitetura de Execução — 1/2

Como Python Executa seu Código

Python usa um sistema híbrido que combina compilador e interpretador:

1
Código-fonte (.py)
Você escreve o código Python legível por humanos em arquivos .py.
2
Compilador → Bytecode (.pyc)
O compilador converte o código-fonte em bytecode — um código intermediário portátil, não específico para nenhuma plataforma.
Arquitetura de Execução — 2/2

Como Python Executa seu Código

3
Máquina Virtual Python (PVM)
A PVM lê o bytecode e o traduz para o código de máquina do sistema operacional em tempo de execução — tornando o código portátil.

Essa portabilidade permite que o mesmo bytecode rode em qualquer SO com a PVM instalada.

Dados na Memória

Variáveis e Tipos Numéricos

Variáveis são "apelidos" para endereços de memória. Python usa tipagem dinâmica — o tipo é inferido automaticamente no momento da atribuição.

tipos_numericos.py
# int — inteiros (sem limite de tamanho) idade = 25 populacao = 8_100_000_000 # float — ponto flutuante pi = 3.14159 temperatura = -3.5 # bool — True ou False ativo = True print(type(idade)) # <class 'int'>
Coleções e Textos — 1/2

Tipos Sequenciais (Conceito)

  • str (strings): Sequências de caracteres, usadas para textos. Imutáveis. Podem ser delimitadas por aspas simples, duplas ou triplas.
  • list (listas): Sequências mutáveis de itens. Permitem duplicatas e geralmente armazenam itens homogêneos.
  • tuple (tuplas): Sequências imutáveis. Geralmente armazenam itens heterogêneos e são mais rápidas que listas.
Coleções e Textos — 2/2

Tipos Sequenciais (Conceito)

  • range: Representa uma sequência imutável de números. Frequentemente usada para controlar o número de repetições em loops for.
  • dict (dicionários): Coleções de pares chave-valor, onde as chaves são definidas pelo usuário e são obrigatoriamente únicas.

Diferente das sequências, os dicionários não são acessados por índices numéricos, mas sim pelas suas chaves.

Lógica e Cálculo — 1/2

Operadores do Python

  • Aritméticos: Realizam cálculos matemáticos. Além dos básicos (+, -, *, /), inclui divisão inteira (//), resto (%) e exponenciação (**).
  • Comparação: Comparam valores e retornam um booleano (True ou False). Exemplos: ==, !=, >, <, >=, <=.

Operadores do Python

  • Atribuição: Atribuem valores a variáveis. Podem ser combinados com aritméticos (+=, -=, *=) para simplificar atualizações de valores.

Paradigma da Programação Estruturada

O fim do código macarrônico através dos três pilares fundamentais.

Histórico e Lógica

Programação Estruturada

Criado na década de 1950, este paradigma defende que qualquer programa pode ser escrito usando apenas três estruturas de controle:

Sequência
Seleção (Condicional)
Iteração (Repetição)
Receita de bolo: O fluxo segue uma ordem clara (ingredientes → mistura → forno). Tudo com um fluxo lógico e previsível.
Os 3 Pilares — 1/3

Pilar 1: Sequência

As instruções são executadas uma após a outra, na ordem em que aparecem no código. É o fluxo padrão de qualquer programa.

INÍCIO Leia A Leia B C ← A + B Mostre C FIM
a = 5 b = 10 c = a + b print(c)
Os 3 Pilares — 2/3

Pilar 2: Seleção

Permite que o programa tome decisões e siga caminhos diferentes com base em condições booleanas (if, elif, else).

if nota >= 7: print("Aprovado") else: print("Reprovado")
Os 3 Pilares — 3/3

Pilar 3: Iteração

Permite repetir um bloco de código várias vezes enquanto uma condição for verdadeira (while) ou sobre uma sequência (for).

# Repetir 5 vezes for i in range(5): print(f"Contagem: {i}")
Por que usar?

Vantagens da P. Estruturada

  • Fácil Entendimento: O fluxo linear é mais simples de ler.
  • Depuração: Erros são mais fáceis de localizar.
  • Manutenibilidade: Código modular e organizado.
  • Redução de Complexidade: Elimina o "código macarrônico".

Orientação a Objetos (POO)

Encapsulamento, Herança, Polimorfismo e Abstração.

Um Novo Paradigma

A Arte dos Objetos

Enquanto a estruturada foca em "o que fazer" (verbos/ações), a POO foca em "quem faz" (substantivos/objetos). Um objeto combina dados e comportamentos.

Controle Remoto: É um objeto com atributos (cor, marca) e métodos (ligar, mudar canal). Você interage com ele sem precisar saber como o circuito interno funciona.
Os 4 Pilares da POO — 1/4

1. Encapsulamento

Agrupa dados e métodos em uma unidade (classe) e restringe o acesso direto a eles, protegendo contra modificações acidentais.

class Conta: def __init__(self, saldo): self.__saldo = saldo # Privado
Os 4 Pilares da POO — 2/4

2. Herança

Permite que uma classe filha herde atributos e métodos de uma classe pai, promovendo o reuso de código.

class Carro(Veiculo): pass # Herda tudo de Veiculo
Os 4 Pilares da POO — 3/4

3. Polimorfismo

Permite que diferentes objetos respondam à mesma mensagem de formas distintas (ex: o método falar() pode latir ou miar).

animais = [Cão(), Gato()] for a in animais: a.falar() # Au au / Miau
Os 4 Pilares da POO — 4/4

4. Abstração

Foca apenas nas características essenciais de um objeto para o problema, ocultando detalhes complexos ou irrelevantes.

Exemplo: Para um sistema de RH, o objeto Pessoa precisa de CPF. Para um jogo, apenas de nome e vida.
