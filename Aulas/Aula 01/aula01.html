Aula 01: Onboarding e Ambiente

Início da Jornada

Momento Inicial:

  • Apresentação do Professor.
  • Apresentação da Unidade Curricular.
  • Definição da estratégia de aprendizado.

Estratégia: "Back-End First"

Explicar a estratégia de focar primeiro no funcionamento lógico e dados e só depois na aparência.

1º Foco (Lógica/Dados):

  • Python
  • SQL
  • Blocos 1, 2 e 3

2º Foco (Interface):

  • HTML / CSS
  • "Ligar os fios"
  • Bloco 4

Por que Back-End First?

Objetivo: Reduzir a carga cognitiva.

Primeiro garante-se que o sistema salva e processa dados corretamente.

Ao final, o trabalho será apenas conectar a lógica já funcional a uma interface web, facilitando o entendimento do fluxo completo.

1. Contextualização: A Situação

O objetivo é dar sentido ao curso, transformando a sala de aula em um ambiente corporativo.

Cenário da Empresa:

Vocês atuarão como técnicos contratados em uma empresa de tecnologia especializada em soluções industriais que está em fase de expansão.

A Missão: O setor de engenharia de software (a turma) recebeu a encomenda de desenvolver novos sistemas para automatizar processos de diversos clientes.

2. Definição de Times e Produtos

Vocês devem se organizar em "squads" (equipes) e escolher um dos produtos do portfólio da empresa.

Catálogo de Produtos (Parte 1):

  • Delivery de comida
  • Aplicativo de transporte
  • Loja de departamentos
  • Streaming de vídeo
  • Gerenciamento de estoque

Catálogo de Produtos (Parte 2)

Continuação das opções disponíveis para os Squads:

  • Hospital
  • Escola
  • Empresa de Recursos Humanos (RH)
  • Oficina mecânica
  • Fábrica de chocolate

Nota: Cada projeto tem particularidades, mas todos exigirão banco de dados, lógica de programação e interface.

3. Configuração do Ambiente

Preparação das ferramentas de trabalho listadas no Plano de Ensino.

Instalação/Verificação:

  • Garantir acesso ao VS Code.
  • Garantir acesso ao Git.

Configuração do Git:

  • Configurar user.name e user.email no terminal.
  • Criar conta no GitHub se necessário.

4. Desafio: "Hello World da Empresa"

Atividade prática valendo como avaliação formativa inicial.

Objetivo: Configurar seu ambiente profissional, criar o repositório oficial e realizar o primeiro "commit".

Cenário: Você acabou de ser contratado e precisa preparar sua estação de trabalho para iniciar o desenvolvimento do sistema escolhido (ex: Hospital, Delivery).

Passo 1: Definição do Tema

Antes de codificar, seu grupo deve ter escolhido um dos 10 temas do portfólio.

O "Produto":

É fundamental que todo o grupo esteja alinhado com o tema escolhido.

Exemplos:

  • Delivery
  • Hospital
  • Escola
  • Oficina Mecânica, etc.

Passo 2: Preparação do Git

Abra o terminal (ou VS Code) e configure sua identidade. Isso garante que a empresa saiba "quem fez o quê".

Digite os comandos (substitua pelos seus dados):

git config --global user.name "Seu Nome Completo"

git config --global user.email "seu.email@exemplo.com"

Passo 3: A "Nuvem" (GitHub)

  1. Acesse o GitHub (ou plataforma da instituição).
  2. Crie um Novo Repositório.

Padronização: Nomeie conforme o tema (ex: sistema-hospital, app-delivery).

Mantenha como Público e copie a URL gerada (ex: https://github.com/usuario/sistema.git).

Passo 4: Clonagem

Trazendo o repositório para sua máquina local.

No seu computador, crie uma pasta para organizar projetos e execute:

git clone https://github.com/seu-usuario/nome-do-projeto.git

Entre na pasta criada:

cd nome-do-projeto

Passo 5: Main.py

Abra a pasta no VS Code e crie o arquivo main.py com o seguinte código (adapte ao tema):

# Arquivo principal do sistema

print("Sistema de Gestão Hospitalar Iniciado")

# ou

print("App de Delivery - Módulo Backend Iniciado")

Nota: O foco agora é apenas garantir que o Python roda e o ambiente está configurado, sem telas por enquanto.

Passo 6: Versionamento e Envio

Execute a sequência sagrada do Git no terminal do VS Code:

1. Adicionar à caixa de envio:
git add .

2. Selar a caixa (Commit):
git commit -m "Primeiro commit: Estrutura inicial"

3. Enviar para a nuvem:
git push

Passo 7: Critério de Sucesso

Para considerar a atividade concluída com êxito:

  1. Atualize a página do GitHub no navegador.
  2. Verifique se o arquivo main.py aparece na lista.
  3. Clique no arquivo e confirme se o código print("...") está lá.

Checklist de Verificação

  • [ ] O VS Code está instalado e aberto na pasta correta?
  • [ ] O repositório foi clonado (não apenas pasta manual)?
  • [ ] O arquivo chama-se main.py?
  • [ ] Rodar python main.py exibe a mensagem correta?
  • [ ] O GitHub foi atualizado com o último commit?

5. Encerramento

Verificação Final:

Verificar nos repositórios se o arquivo .py existe e se o histórico de commits foi iniciado.
