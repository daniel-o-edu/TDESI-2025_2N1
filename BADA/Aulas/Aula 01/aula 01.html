2.1 O que é um Banco de Dados?

Definição Simples: Uma coleção organizada de dados que se relacionam, criada para preservar, gerenciar e entregar informações de forma eficiente.

Analogia Ineficiente:

Arquivo de gavetas desorganizado (difícil de achar).

Analogia Eficiente:

Sistema digital indexado (busca instantânea).

2.2 Dado vs. Informação - Parte 1

Este é o conceito mais importante da noite!

Dado (Matéria-prima)

É definido como um registro bruto, destituído de contexto.

  • O dado por si só não permite tomada de decisão.
  • Exemplos: "38", "Azul" ou "12/05".
  • Sozinhos, eles são apenas caracteres.

2.2 Dado vs. Informação - Parte 2

Informação (Produto final)

É o resultado do processamento e organização do dado, atribuindo-lhe significado útil.

Exemplo: Quando o dado "38" é contextualizado como "A temperatura do paciente é 38 graus", ele se torna uma informação (indica febre).

O objetivo de um banco de dados é justamente preservar o dado para entregar essa informação de forma eficiente.

3.1 Definição e Propósito

Muitas vezes confundido com um simples "lugar para guardar arquivos", o conceito é mais refinado:

  • Organização e Relacionamento: É uma coleção organizada onde os dados se relacionam entre si. Não é um depósito aleatório.
  • Eficiência: Criado para gerenciar e entregar informações de maneira eficiente.
  • Busca: Funciona como um sistema digital indexado para busca instantânea.

3.2 Evolução: Planilhas vs. SGBDs

A transição do Excel para Sistemas Gerenciadores de Banco de Dados (SGBDs) é motivada por limitações críticas em ambientes corporativos:

Característica Planilhas (Excel) Banco de Dados (SGBD)
Acesso Geralmente monousuário Multiusuário simultâneo
Segurança Baixa (fácil copiar) Alta (permissões granulares)
Volume Lento com muitos dados Milhões/Bilhões de registros

3.2 Evolução (Continuação)

Característica Planilhas (Excel) Banco de Dados (SGBD)
Integridade Aceita erros (texto em data) Regras estritas de validação
Centralização Arquivos espalhados (v1, v2) Centralizado e único

O SGBD resolve a redundância e a falta de controle típicas de sistemas baseados em arquivos/planilhas.
