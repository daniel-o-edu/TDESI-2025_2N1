Aula 11 - Instalação e Configuração do SGBD

Descrição: Realizar a instalação e configuração do SGBD MySQL e a ferramenta Workbench. Criar o banco de dados e as primeiras tabelas. Hoje vamos realizar uma aula prática, com a configuração inicial do SGBD MySQL Workbench que passaremos a utilizar em nossas atividades e no nosso projeto de SA.

CONFIGURAÇÃO INICIAL

1°. Buscar no Windows 11: Abrir o menu Iniciar do Windows 11 e buscar por "MySQL Workbench".

2º. Gestão de Serviços (Troubleshooting): Abrir o services.msc no Windows caso o banco não esteja rodando, procurando pelo serviço MySQL80 (ou similar) e garantindo que o status seja "Em Execução".

CONFIGURAÇÃO INICIAL (continuação)

3°. Criando a Conexão:

  • Clicar no ícone + ao lado de MySQL Connections.
  • Configurar o Connection Name (ex: "Conexao_Local").
  • Manter o Hostname como 127.0.0.1 e a Port como 3306.
  • Configurar o Username (geralmente root).
  • Clicar em Test Connection, criar a senha e marcar a opção "Save password in vault" para facilitar os próximos acessos.

NAVEGAÇÃO E RECONHECIMENTO DA INTERFACE

Dominar o MySQL Workbench é o primeiro passo para se tornar um mestre em bancos de dados. Ele é a nossa "central de controle", onde visualizamos e manipulamos tudo o que acontece no servidor.

Vamos explorar as áreas principais:

  • - Painel Lateral (Navigator)

O Navigator é o seu mapa. Ele é dividido em duas abas principais na parte inferior, e entender a diferença entre elas é fundamental.

1. Aba Management (Administração)

Focada na saúde e no controle do servidor.

  • Server Status: Mostra se o banco está ligado, consumo de CPU e conexões ativas.
  • Users and Privileges: Onde criamos novos usuários e definimos o que cada um pode fazer.
  • Data Export/Import: Ferramentas cruciais para fazer backup (Export) ou restaurar um banco de dados (Import).

2. Aba Schemas (Esquemas / Bancos de Dados)

É aqui que passaremos 90% do nosso tempo.

  • Exibe todos os bancos de dados criados.
  • Permite expandir cada banco para ver suas Tables (Tabelas), Views (Visões), Stored Procedures e Functions.

NAVEGAÇÃO E RECONHECIMENTO DA INTERFACE (continuação)

- Área de Script (Query Editor)

A área de script (ou Query Editor) é o coração do MySQL Workbench. É nela que você escreve as instruções que o banco de dados deve seguir. Para utilizá-la com eficiência, pense nela como um editor de texto especializado que "fala" com o servidor.

Aqui está o passo a passo de como tirar o melhor proveito:

1o. Preparando o Ambiente

Antes de começar a digitar, você precisa dizer ao Workbench em qual banco de dados deseja trabalhar.

  • No Painel Schemas, dê um duplo clique sobre o nome do banco de dados (ex: meu_banco).
  • O nome ficará em negrito, indicando que ele é o banco padrão. Agora, qualquer comando que você digitar será executado ali dentro.

2o. Escrevendo o Código

O editor possui recursos que facilitam sua vida:

  • Syntax Highlighting: O Workbench colore as palavras-chave (como SELECT, FROM, INSERT) em azul para ajudar na leitura.
  • Auto-complete: Se você começar a digitar uma tabela que já existe, ele sugerirá o nome. Aperte Tab para completar.

3o. Executando os Comandos (O Momento da Verdade)

Escrever o código não faz nada sozinho; você precisa "enviar" a instrução para o servidor usando os ícones de raio na barra de ferramentas superior:

  • O Raio com Cursor (Preferido): Coloque a barra de digitação (cursor) em qualquer lugar da linha que você quer rodar e clique neste botão. Ele executa apenas aquela instrução específica.
  • O Raio sem Cursor: Ele vai ler o seu arquivo de cima a baixo. Se houver 50 comandos, ele tentará rodar os 50. Use com cuidado!

Dica de Ouro: Atalho de Teclado

Para ganhar agilidade, em vez de clicar nos botões de raio, você pode usar:

  • Ctrl + Enter: Executa a linha onde o cursor está (equivale ao raio com o cursor).
  • Ctrl + Shift + Enter: Executa o script inteiro (equivale ao raio limpo).

Com essas ferramentas, você já consegue navegar, criar e corrigir suas consultas com total controle!

4o. Interpretando os Resultados

Assim que você executa um comando, duas coisas acontecem:

  1. Painel de Saída (Output): Verifique se apareceu o ícone verde (sucesso).
  2. Result Grid: Se você fez uma consulta (SELECT), uma nova aba aparecerá logo abaixo do seu código mostrando os dados em formato de planilha.

5o. Organização e Salvamento

Você pode abrir várias abas de script clicando no ícone de "+" sobre uma pasta na barra de ferramentas.

  • Se você escreveu um código complexo e quer guardá-lo para amanhã, vá em File > Save Script As.... Ele salvará um arquivo com extensão .sql que você pode abrir em qualquer outro editor de texto.

- Painel de Saída (Output)

Localizado na parte inferior, o Output é o "feedback" do MySQL. Ele te diz em tempo real se o que você fez funcionou ou se você cometeu algum erro de digitação.

  • Bolinha Verde (Success): Significa que o MySQL entendeu seu comando e o executou perfeitamente. Geralmente mostra quantas linhas foram afetadas.
  • Bolinha Vermelha (Error): Ocorreu um erro. O segredo de um bom programador não é não errar, mas sim ler a mensagem de erro.
  • Dica: O código de erro (como o Error Code: 1064) geralmente aponta exatamente onde a sintaxe falhou (perto de qual palavra ou vírgula).

CRIAÇÃO E CONFIGURAÇÃO DO BANCO DE DADOS (Schema)

Para criar o seu primeiro banco de dados no MySQL Workbench, você tem dois caminhos: o "jeito visual" (clicando em botões) ou o "jeito programador" (escrevendo código).

Aqui está o passo a passo para ambos:

Opção 1: O Caminho Visual (Mais fácil para iniciantes)

Na barra de ferramentas superior, logo abaixo dos menus principais, você verá uma série de ícones.

  1. Procure o ícone que parece um cilindro (moeda) com um símbolo de "+" ou um brilho (chamado "Create a new schema in the connected server").
  2. Ao clicar nele, abrirá uma aba chamada "New Schema".
  3. No campo Name, digite o nome do seu banco (ex: loja_virtual).

Dica: Evite espaços ou acentos em nomes de bancos de dados.

Opção 1 (continuação)

  1. Clique no botão Apply no canto inferior direito.
  2. Uma janela aparecerá mostrando o código SQL que o Workbench criou para você. Clique em Apply novamente e depois em Finish.

Opção 2: O Caminho via Script (O "Jeito Programador")

Se você já estiver com a área de script aberta, pode fazer tudo via teclado:

  1. Digite o seguinte comando:
CREATE DATABASE nome_do_seu_banco;
  1. Mantenha o cursor na linha desse comando e aperte Ctrl + Enter (ou clique no raio com o cursor).
  2. Verifique no Painel de Saída (Output) se apareceu a bolinha verde.

O Passo Final Importante: O Refresh

Às vezes, após criar o banco, ele não aparece imediatamente na lista da esquerda (Painel Schemas).

  • Vá até o painel Schemas.
  • Clique no ícone de duas setinhas em círculo (Refresh) no canto superior direito do painel.
  • Pronto! Seu novo banco aparecerá na lista.

Lembre-se: Após criar o banco, dê um duplo clique sobre o nome dele para deixá-lo em negrito. Isso avisa ao Workbench que todos os próximos comandos (como criar tabelas) devem ser feitos dentro desse banco específico.

CRIANDO AS TABELAS

Agora que você já tem o seu banco de dados (o "container"), é hora de criar as tabelas, que é onde os dados realmente moram. Imagine que o banco de dados é um arquivo e as tabelas são as planilhas dentro dele.

Conceitos Fundamentais (Os "Ingredientes")

Antes de colocar a mão na massa, você precisa entender quatro conceitos que definem como cada coluna da sua tabela vai se comportar:

  • Chave Primária (PK - Primary Key): É o "RG" da linha. Cada registro na tabela deve ter um identificador único para que nunca existam dois itens iguais.
  • Auto Incremento (AI): Faz com que o banco de dados gere o próximo número automaticamente (1, 2, 3...). Você não precisa digitar o ID; o MySQL faz isso por você.

Tipos de Dados:

  • INT: Números inteiros (ex: idade, quantidade).
  • VARCHAR(X): Textos de tamanho variável. O X é o limite de caracteres (ex: um nome pode ter VARCHAR(100)).
  • DATE: Para datas no formato AAAA-MM-DD.
  • DECIMAL(10,2): Para valores monetários (o 10 é o total de dígitos e o 2 são as casas decimais).

Criando a Tabela 1: Via Interface Visual

Essa forma é excelente para quem está começando, pois o Workbench te guia através de campos clicáveis.

  1. No painel Schemas, localize o seu banco de dados e clique na setinha ao lado dele.
  2. Clique com o botão direito em Tables e selecione Create Table....
  3. No campo Table Name, digite clientes.

Criando a Tabela 1 (continuação)

4. Na área de colunas abaixo, vamos configurar os campos:

  • id: Tipo INT. Marque os campos PK (Chave Primária), NN (Not Null - não pode ser vazio) e AI (Auto Incremento).
  • nome: Tipo VARCHAR(100). Marque NN.
  • data_nascimento: Tipo DATE.

5. Clique no botão Apply.

Atenção: Antes de o Workbench criar a tabela, ele abrirá uma janela com o código SQL. Leia esse código! Ele é exatamente o que o banco de dados vai executar por trás dos panos.

Clique em Apply e depois em Finish.

Criando a Tabela 2: Via Script (O Jeito DDL)

Agora que você viu como a interface funciona, vamos fazer "na raça" usando comandos de DDL (Data Definition Language). Vamos criar uma tabela de produtos como exemplo.

1. Abra uma nova aba de script (Ctrl + T).

2. Digite o comando abaixo:

CREATE TABLE produtos (
id_produto INT PRIMARY KEY AUTO_INCREMENT,
nome_produto VARCHAR(45) NOT NULL,
preco DECIMAL(10,2) NOT NULL,
estoque INT DEFAULT 0
);

3. Coloque o cursor em qualquer lugar deste código e aperte Ctrl + Enter.

4. Confira a bolinha verde no Painel de Saída.

Por que aprender os dois jeitos?

A interface visual é ótima para prototipar rápido e evitar erros de digitação. Já o script é o que você usará no dia a dia como desenvolvedor, pois permite salvar o código, versionar no Git e automatizar a criação do banco em outros servidores.

Você conseguiu visualizar a sua primeira tabela na lista à esquerda depois de dar um Refresh?

VISUALIZAR O DER

Visualizar o banco de dados como um DER (Diagrama Entidade-Relacionamento) é um dos recursos mais poderosos do MySQL Workbench. Isso transforma o seu código em um mapa visual, facilitando o entendimento de como as tabelas se conectam.

No Workbench, esse processo é chamado de Reverse Engineering (Engenharia Reversa).

Passo a Passo para Gerar o Diagrama

  1. Acesse a Ferramenta: No menu superior do MySQL Workbench, clique em Database e depois em Reverse Engineer... (ou use o atalho Ctrl + R).
  2. Conexão: Uma janela abrirá perguntando sobre a conexão com o servidor. Se você já estiver conectado, basta clicar em Next.
  3. Execução: O Workbench se conectará ao banco para ler as informações. Clique em Next novamente após ele concluir os testes.

Passo a Passo para Gerar o Diagrama (continuação)

  1. Seleção do Banco: Agora, selecione o Schema (o banco de dados) que você criou e deseja visualizar. Clique em Next.
  2. Importação: Ele lerá os objetos (tabelas, views, etc.). Clique em Next e, na última tela, em Execute.
  3. Finalização: Clique em Next e depois em Finish.

O Que Você Verá na Tela (Model Viewer)

O Workbench abrirá uma nova aba chamada "MySQL Model". Nela, você verá suas tabelas representadas por quadros:

  • As Colunas: Todos os nomes e tipos de dados estarão visíveis.
  • As Chaves: Você verá um ícone de chave dourada ao lado dos campos que são PK (Chave Primária).
  • Relacionamentos: Se você criou chaves estrangeiras (Foreign Keys), o Workbench desenhará linhas conectando as tabelas automaticamente.

Por que isso é útil?

  • Documentação: Você pode exportar esse diagrama como uma imagem (File > Export > Export as PNG) para colocar em um trabalho ou apresentação.
  • Organização: Você pode arrastar as tabelas para organizar o layout e entender melhor a arquitetura do seu sistema.
  • Edição Visual: Se você der dois cliques em uma tabela dentro do diagrama, pode alterar colunas ou cores ali mesmo, e depois "sincronizar" de volta para o banco de dados real.

Dica de Ouro: O Atalho Visual

Se você quiser apenas ver as colunas e os índices de uma tabela rapidamente sem gerar o DER completo, você pode clicar no ícone de ferramenta (chave inglesa) que aparece ao passar o mouse sobre o nome da tabela no painel lateral (Schemas).

Conseguiu gerar o desenho das suas tabelas? Se elas não estiverem conectadas por uma linha, é porque ainda não criamos um relacionamento formal entre elas!

EXPORTAR O BD

Exportar um banco de dados (o famoso Backup ou Dump) é essencial para levar seu trabalho para outro computador ou para enviar ao seu professor. No MySQL Workbench, fazemos isso através da aba Administration.

1. Onde encontrar a ferramenta

  1. No painel lateral esquerdo (Navigator), clique na aba inferior chamada Administration.
  2. Procure a opção Data Export e clique nela.

2. Configurando a Exportação

Uma nova aba se abrirá com várias opções. Siga esta ordem:

  1. Selecionar o Banco: Na lista da esquerda ("Tables to Export"), marque o quadradinho do banco de dados que você quer exportar.
  2. Selecionar Tabelas: À direita, você pode escolher exportar todas as tabelas (padrão) ou apenas algumas específicas.

3. Opções de Exportação:

Logo abaixo, você verá duas opções principais:

  • Export to Dump Project Folder: Cria uma pasta com vários arquivos separados (bom para projetos gigantes).
  • Export to Self-Contained File (Recomendado): Esta é a melhor opção para estudantes. Ela cria um único arquivo .sql que contém tudo: a criação das tabelas e os dados que estão dentro delas.

3. Detalhes Importantes (Checklist)

Antes de clicar no botão final, verifique dois detalhes no menu suspenso acima do botão de exportar:

  • Dump Structure and Data: Exporta o desenho das tabelas E os dados que você inseriu. (Geralmente é o que você quer).
  • Dump Structure Only: Exporta apenas o "esqueleto" (as tabelas vazias).
  • Dump Data Only: Exporta apenas os registros, sem criar as tabelas.

4. Finalizando

  1. No campo Export to Self-Contained File, clique nos três pontinhos ... para escolher onde salvar o arquivo no seu computador e dê um nome a ele (ex: backup_aula_01.sql).
  2. Clique no botão Start Export no canto inferior direito.
  3. Se o Workbench pedir sua senha do banco de dados, digite-a.
  4. Quando a barra de progresso chegar ao fim e ficar verde, seu arquivo está pronto!

Como importar depois?

Se você precisar abrir esse arquivo em outro computador:

  1. Vá em Administration > Data Import/Restore.
  2. Selecione Import from Self-Contained File e localize seu arquivo .sql.
  3. Escolha o banco de destino (ou crie um novo em New...) e clique em Start Import.

Dica de ouro: Sempre verifique se o arquivo .sql foi gerado com um tamanho maior que 0kb. Se ele estiver vazio, algo deu errado na permissão de pasta ou na seleção das tabelas!

ATIVIDADE

Agora que você possui todas as informações e ferramentas para o Banco de Dados do seu projeto de SA, vamos iniciar os trabalhos. Utilizando o MER elaborado na última aula antes da Avaliação Objetiva (Aula 09) vamos criar o Banco de Dados do projeto.

1. Criando o Banco de Dados

  • Crie o BD com o nome do projeto, lembrando das boas práticas (nomes sem espaços, uso de underscores, padrões de nomenclatura).
  • Crie TODAS as tabelas, com suas respectivas chaves primárias e atributos.
  • Revisar se os tipos de dados (VARCHAR, INT, DECIMAL) fazem sentido para cada atributo do MER original.

2. Montando o DER

  • Após a criação das tabelas, você deve executar o processo de criação do DER, conforme descrito nesta aula.
  • Relacionamentos: Se o MER tinha relacionamentos (1:N, N:M), lembre-os de que no DER isso deve aparecer através das Foreign Keys (FK). Sem as FKs, o diagrama gerado pelo Workbench mostrará apenas tabelas "soltas" no espaço.
  • Com o DER criado, exporte como arquivo png.

3. Finalizando

  • Após todo processo realizado, exporte o Banco de Dados criado e o arquivo png do DER, commitando eles no repositório do seu projeto no GitHub na pasta “Backend”.
  • Atenção ao Export: Dica para exportar com a opção "Self-Contained File". Se vocês exportarem como "Project Folder", o GitHub ficará com dezenas de arquivinhos pequenos, o que dificulta a leitura do repositório.

Atenção: Estou disponibilizando dois Bancos de Dados para estudo, treinamento e testes.

SAKILA

O banco de dados Sakila é um esquema relacional de exemplo amplamente utilizado, criado para simular uma locadora de DVDs/filmes. Ele é ideal para prática de SQL, tutoriais e testes de recursos do MySQL, como Views, Triggers e Stored Procedures. O modelo inclui tabelas para filmes, atores, clientes, aluguéis e pagamentos.

SAKILA - Principais Características e Usos

  • Finalidade: Criado originalmente por Mike Hillyer (ex-MySQL) para livros e tutoriais.
  • Estrutura: Bem normalizado, facilitando o aprendizado de joins, agregrações e modelagem de dados.
  • Conteúdo: Contém tabelas de atores, endereços, categorias, cidades, países e tabelas centrais de estoque/locação.

SAKILA - Instalação e Exemplos

  • Instalação: Frequentemente usado no MySQL Workbench para demonstrar consultas reais.
  • Exemplos de consultas (Uso):
    • Listar todos os filmes de um determinado ator.
    • Calcular a receita total de uma loja.
    • Identificar clientes inativos ou com pagamentos pendentes.

WORLD

O banco de dados de exemplo World é um clássico da MySQL para estudos, contendo dados geográficos e demográficos, como países, cidades e idiomas.

Ele é ideal para praticar consultas SQL de leitura, possuindo três tabelas principais: country, city e countrylanguage.
