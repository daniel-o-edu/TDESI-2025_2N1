Conceitos Fundamentais

Try/Except, Testes Unitários e Depuração.

Seção 2.1

Conceito 1: Tratamento de Exceções (try / except)

Uma exceção é um evento que ocorre durante a execução de um programa e que interrompe o fluxo normal das instruções (por exemplo, tentar dividir um número por zero ou tentar converter a palavra "dez" em um número inteiro).

Em Python, quando o interpretador encontra um erro desse tipo, ele levanta (raise) uma exceção. Se essa exceção não for tratada, o programa é encerrado com uma mensagem de erro no terminal.

Para evitar isso, utilizamos a estrutura try/except (tentar/exceto). O código que tem potencial para dar erro é colocado dentro do bloco try.

Seção 2.1

Como Funciona a Recuperação

Se um erro ocorrer lá dentro, o programa não quebra; em vez disso, ele pula imediatamente para o bloco except, onde podemos lidar com o erro de forma amigável, alertar o usuário e permitir que a execução continue normalmente.

Relevância
Essa prática é essencial para garantir a disponibilidade e a confiabilidade dos sistemas modernos.
Seção 2.1 · Demonstração

Solicitando Valor de Forma Segura

# Solicitando um valor financeiro ao usuário de forma segura while True: try: # Tenta converter a entrada diretamente para float valor = float(input("Digite o valor do depósito: R$ ")) print(f"Depósito de R$ {valor:.2f} realizado com sucesso!") break # Sai do loop se der tudo certo except ValueError: # Se o usuário digitar "cem", o float() levanta um ValueError # O programa cai aqui em vez de crashar print("ERRO: Por favor, digite apenas valores numéricos válidos (ex: 150.50).")
Seção 2.1 · Explicação

Explicação linha por linha

  • while True: Cria um loop infinito para insistir até que o usuário acerte.
  • try: Inicia o bloco de "zona de risco", onde o erro pode ocorrer.
  • valor = float(...): Tenta converter o texto em um número decimal. Se o usuário digitar "abc", o Python entra em pânico e lança a exceção ValueError.
  • break: Se a linha acima não der erro, o break é executado, encerrando o while.
  • except ValueError: Captura especificamente o erro de conversão de valor. Em vez de fechar o programa, ele executa o bloco abaixo.
  • print(...): Exibe uma mensagem amigável instruindo o usuário a corrigir o erro.
Seção 2.1 · Exercício

Exercício Guiado

Desafio

Use try/except para capturar o erro ZeroDivisionError caso o usuário tente dividir por zero, avisando-o que "A divisão por zero não é permitida matematicamente".

Seção 2.2

Conceito 2: Testes Unitários Automatizados (unittest)

Testar um programa manualmente (clicando em botões ou digitando valores no terminal toda vez que mudamos uma linha de código) é insustentável em grandes projetos, consome muito tempo e é sujeito a falhas humanas.

No Mercado
No mercado corporativo, os desenvolvedores escrevem "código que testa código".

O Teste Unitário verifica o menor pedaço testável de um software (geralmente uma função ou método isolado) para garantir que ele retorna exatamente o que foi projetado para retornar.

Seção 2.2

A biblioteca unittest

O Python possui uma biblioteca nativa chamada unittest. Com ela, criamos métodos de teste que utilizam asserções (como assertEqual, que verifica se "A" é igual a "B").

Se alterarmos a regra de negócios no futuro e a função passar a calcular errado, o teste automatizado falhará imediatamente, nos alertando sobre o defeito antes que o software seja entregue ao cliente.

A biblioteca unittest é um framework de testes unitários embutido na biblioteca padrão do Python. Inspirada no JUnit do Java, ela fornece um conjunto robusto de ferramentas para construir, executar e relatar testes em seu código, garantindo que ele funcione como esperado.

Seção 2.2

Funcionalidades e Estruturas

A. Casos de Teste (unittest.TestCase)

Para criar testes, você deve criar uma classe que herde de unittest.TestCase. Dentro dessa classe, qualquer método cujo nome comece com a palavra test_ será automaticamente reconhecido e executado como um teste.

B. Asserções (Assertions)

As asserções são os métodos usados dentro dos testes para verificar se o resultado obtido é igual ao resultado esperado. Se a asserção falhar, o teste falha. As principais são:

  • assertEqual(a, b): Verifica se a == b.
  • assertNotEqual(a, b): Verifica se a != b.
  • assertTrue(x): Verifica se x é verdadeiro.
Seção 2.2

Mais Asserções

  • assertFalse(x): Verifica se x é falso.
  • assertIsNone(x): Verifica se x é None.
  • assertIn(a, b): Verifica se a está contido em b (como em listas ou strings).
  • assertRaises(Excecao, funcao, *args, **kwargs): Verifica se uma função levanta (dispara) uma exceção específica quando chamada com os argumentos fornecidos.
Seção 2.2

C. Fixtures (Preparação e Limpeza)

Muitas vezes, você precisa configurar um ambiente antes de testar (como conectar a um banco de dados temporário) e limpá-lo depois. O unittest fornece métodos especiais para isso:

  • setUp(): Executado antes de cada método de teste individual. Ideal para criar variáveis ou estados iniciais.
  • tearDown(): Executado depois de cada método de teste. Ideal para fechar conexões ou limpar arquivos temporários.
  • setUpClass() / tearDownClass(): Métodos de classe (decorados com @classmethod) executados apenas uma vez antes e depois de todos os testes daquela classe, respectivamente.
Seção 2.2

D. Executando e E. Mocking

D. Executando os Testes (unittest.main())

A maneira mais simples de executar os testes dentro de um arquivo script é chamar a função principal da biblioteca no final do arquivo:

if __name__ == '__main__': unittest.main()

Isso varre o arquivo atual, encontra todos os casos de teste e os executa, imprimindo um relatório no console.

E. Mocking (unittest.mock)

Embora seja um módulo à parte, o unittest.mock é frequentemente usado em conjunto. Ele permite que você substitua partes do seu sistema (como chamadas de API externas ou funções complexas) por objetos "falsos" (mocks) para que você possa testar seu código de forma isolada, sem depender de recursos externos.

Demonstração: Código Base

Arquivo
matematica.py (O código a ser testado)
def calcular_dobro(numero): return numero * 2
Seção 2.2 · Demonstração

Demonstração: Script de Teste

Arquivo
teste_matematica.py (O script de teste)
import unittest from matematica import calcular_dobro class TestesMatematica (unittest.TestCase): def test_calcular_dobro_positivo(self): # A função assertEqual verifica se o 1º argumento é IGUAL ao 2º # Esperamos que o dobro de 5 seja exatamente 10 self.assertEqual(calcular_dobro(5), 10) def test_calcular_dobro_negativo(self): # Testando como a função lida com números negativos self.assertEqual(calcular_dobro(-3), -6) if __name__ == '__main__': unittest.main()
Seção 2.2 · Explicação

Explicação linha por linha

  • import unittest: Traz o módulo de testes do Python para o nosso arquivo.
  • class TestesMatematica (unittest.TestCase): Cria um grupo de testes que herda as capacidades do módulo unittest.
  • def test_calcular_dobro_positivo(self): Todo método que testa algo OBRIGATORIAMENTE deve começar com a palavra test_.
  • self.assertEqual(...): Executa a função passando 5 e verifica se a resposta interna do Python é 10. Se não for, o teste grita que falhou.
  • unittest.main(): É o motor que procura todos os métodos com test_ e os executa de uma vez, gerando um relatório no console.

Seção 2.3

Conceito 3: Depuração de Código (Debug e Breakpoints)

O processo de encontrar e remover falhas lógicas (bugs) é chamado de depuração (Debugging).

Programadores iniciantes costumam encher o código de funções print() para tentar descobrir onde uma variável está com o valor errado. Isso suja o código e é ineficiente. A abordagem profissional é utilizar o Debugger (Depurador) integrado à IDE (como VS Code, PyCharm ou o IDLE).

Como Funciona
O Depurador permite que você insira um Breakpoint (ponto de parada) em uma linha específica de código. Quando você executa o programa no modo Debug, o interpretador "congela" a execução exatamente naquela linha, permitindo que você analise o valor atual das variáveis na memória.

Exercício de Fixação: Debugger

Aplicações no Mercado

Como o mundo real utiliza try/except e testes unitários diariamente.

Seção 4

Aplicações no Mercado

  • Integração de APIs e Redes: Aplicações de celular que se comunicam com servidores. Se você abre o app do seu banco dentro do elevador (sem internet) e ele não usa try/except, o aplicativo fecha na sua cara. Empresas utilizam try: para tentar conectar. Se der o erro except ConnectionError:, elas mostram um dinossauro na tela ou uma mensagem fofa: "Parece que você está sem conexão". Isso é controle de danos na experiência do usuário (UX).
  • Pipelines de CI/CD em Equipes: No Nubank ou iFood, quando um programador escreve um código novo e envia para o servidor (GitHub), o código não vai para a versão de produção (para o cliente final) imediatamente. Uma máquina liga automaticamente e roda os scripts de unittest. Se um único assertEqual falhar, o envio é bloqueado, impedindo que um bug chegue ao celular de milhões de usuários.

FAQ - Dúvidas Reais de Alunos

Seção 5

Perguntas Frequentes

A. Eu posso usar um except: sozinho, sem especificar o erro?
Pode, mas é considerado uma das piores práticas de programação (chamado de Bare Except).
O Debugger altera meu código?
Não. O Debugger é apenas um "óculos de visão de raio-X" para a memória RAM do computador. Ele não altera os arquivos .py, apenas observa a execução.
E. Posso colocar breakpoints onde eu quiser?
Sim, inclusive dentro de blocos except para analisar por que o erro estourou, ou dentro de unittests que estejam falhando para entender o que deu errado com os dados enviados.

Conexão com a Situação de Aprendizagem

Seção 6

Como se encaixa no projeto do semestre?

Na construção do sistema da nossa Situação de Aprendizagem, vocês criarão dezenas de rotas e conexões com Bancos de Dados.

Prevenção de Falhas
Se um usuário preencher um formulário na interface tentando injetar um dado corrompido, o backend em Python de vocês precisará aplicar blocos try/except para impedir que toda a aplicação (API) trave.

Além disso, as regras de negócio mais importantes do projeto (como o cálculo do carrinho de compras do cliente ou a validação de estoque) devem obrigatoriamente ser validadas com testes unitários.

Seção 7

Prática de Fixação: Conta Bancária

  • Crie um método que instancie uma Conta, faça um depósito de 100 e use assertEqual para verificar se o conta.saldo é exatamente 100.
  • Crie outro método que faça um depósito de 200, tente sacar 50 e verifique com assertEqual se o saldo final é 150.
Uso do Debugger

Force um erro lógico no seu código de depósito (ex: somar o dobro do valor sem querer). Coloque um breakpoint na função de depositar. Rode em modo Debug, olhe as variáveis no painel, encontre seu erro, desfaça-o e rode o teste automatizado novamente para ver ficar verde (OK).

Seção 7

Resultado Esperado e Sugestões

Resultado Esperado
Ao digitar "teste" no depósito via terminal do main.py, o programa simplesmente não aceita e pede novamente. Ao rodar o teste.py pelo terminal, ele deve mostrar que 2 testes foram executados com sucesso (OK).

Sugestões de solução:

  • Na estrutura while True do seu menu interativo, coloque o try englobando desde o input de valor numérico até a chamada do método conta.depositar(valor). Se o input falhar, ele nem chegará a chamar a função de depósito.
  • Lembre-se de importar corretamente a classe Conta dentro do seu arquivo de testes usando from modelos import Conta (se seguiram a padronização de arquivos da aula passada).
