Aula 06: Estruturas de Decisão e Repetição

Contextualização Estratégica

No ecossistema de desenvolvimento de software, a linearidade é uma exceção. Programas reais precisam reagir a dados dinâmicos, validar entradas de usuários e processar grandes volumes de informações de forma automatizada. Esta aula situa-se no cerne da lógica de programação, transicionando de scripts estáticos para sistemas inteligentes e reativos. Dominar estruturas de decisão e repetição é o divisor de águas entre um codificador iniciante e um desenvolvedor capaz de construir a lógica de negócio de aplicações complexas, como sistemas bancários, motores de busca e controladores industriais.

Resumo do que será desenvolvido

Exploraremos a fundo a sintaxe e a semântica das instruções condicionais if, elif e else, compreendendo como o interpretador Python avalia expressões booleanas e gerencia blocos de código via indentação. Avançaremos para as estruturas de iteração, desmistificando o laço while para repetições baseadas em estados lógicos e o laço for para a varredura eficiente de coleções e sequências de dados. Além disso, refinaremos o controle de execução com as palavras-reservadas break (interrupção abrupta) e continue (salto de iteração), ferramentas essenciais para a otimização de algoritmos e tratamento de exceções em tempo de execução. Todo o conteúdo será aplicado na construção de um subsistema de Caixa Eletrônico, simulando requisitos reais de persistência de sessão e validação de transações financeiras.

Parte 1

Fundamentação e Contextualização: A Anatomia da Decisão

Problema Real de TI (Narrativa)

Você foi contratado por uma Fintech em ascensão para desenvolver o módulo de segurança de um novo terminal de autoatendimento. O requisito é claro: o sistema não pode simplesmente "rodar"; ele deve decidir. Se o saldo for insuficiente, a transação deve ser negada; se a senha estiver errada três vezes, o cartão deve ser bloqueado; se o valor do saque exceder o limite diário, um alerta deve ser gerado. Sem estruturas de decisão, o software seria uma linha reta cega, incapaz de proteger o patrimônio do cliente ou oferecer opções personalizadas. No mercado, chamamos isso de "Lógica de Negócio" (Business Logic), e ela é o que torna o software valioso para as empresas.

Conceituação Técnica Profunda

A estrutura de decisão fundamental em Python é o if. Tecnicamente, ela realiza uma avaliação de curto-circuito em uma expressão booleana. Quando o interpretador encontra um if, ele avalia se a condição resulta em True ou False. Diferente de linguagens como C++ ou Java, o Python não utiliza chaves {} para delimitar blocos; ele utiliza a indentação (espaços em branco). Isso não é apenas estética, é sintaxe obrigatória que força o desenvolvedor a manter o código organizado, seguindo a PEP 8.

Conceituação Técnica Profunda (Cont.)

A expansão dessa estrutura ocorre com o elif (abreviação de else if) e o else. O elif permite testar múltiplas condições exclusivas em sequência, evitando o aninhamento excessivo de if s, o que no mercado chamamos de "Código Macarrônico". O else atua como uma cláusula de salvaguarda, um "senão" que captura qualquer caso que não tenha atendido às condições anteriores. É a rede de segurança de qualquer algoritmo de decisão robusto.

Analogias com Mercado
No dia a dia de um desenvolvedor, escrever um if é como realizar um Deploy condicional. Imagine um ambiente de Integração Contínua (CI/CD): "Se os testes passarem (if tests_pass == True), faça o deploy; senão (else), notifique a equipe no Slack". Outro conceito é o Flagging ou Feature Toggle, onde partes do código só são executadas se uma chave de configuração estiver ativa. Dominar o if é dominar a inteligência estratégica do seu código.
Demonstração Guiada

Vamos visualizar um código simples de validação:

saldo = 1000.00
valor_saque = float(input("Digite o valor do saque: "))
if valor_saque <= 0:
    print("[ERRO] Valor inválido para saque.")
elif valor_saque > saldo:
    print("[ERRO] Saldo insuficiente.")
else:
    saldo -= valor_saque
    print(f"Saque realizado com sucesso! Novo saldo: R$ {saldo:.2f}")
Conexão com SA e Erro Comum

Conexão com SA: Como exemplo, desenvolveremos um sistema de gestão industrial, as estruturas de decisão serão usadas para monitorar sensores. Se a temperatura de uma máquina exceder 80°C, o sistema deve disparar um log de erro e desligar o motor. Sem o if, a fábrica poderia sofrer um acidente por falta de controle lógico.

Erro Comum: Esquecer o sinal de dois pontos (:) ao final da instrução if ou misturar espaços e tabs na indentação. No Python 3, isso gera um IndentationError imediato, impedindo a execução do script.

Mini Estudo de Caso

Um sistema de e-commerce que não utilizava else corretamente permitiu que cupons de desconto fossem aplicados cumulativamente sem validação, gerando prejuízo de milhares de reais em uma "Black Friday" por erro de fluxo lógico.

Aprofundamento Técnico: A Maestria da Repetição

Expansão Conceitual

Entramos agora no domínio da automação. Se o if decide, os laços de repetição (Loops) executam. O laço while é uma estrutura de repetição condicional. Ele executa um bloco de código "enquanto" uma condição for verdadeira. É ideal para situações onde não sabemos previamente quantas vezes a repetição ocorrerá, como manter um sistema ligado até que o usuário clique em "Sair" ou aguardar uma resposta de um servidor que pode demorar.

Expansão Conceitual (Cont.)

Por outro lado, o laço for em Python é, na verdade, um iterador de sequências. Diferente do for tradicional do C (com contador), o for do Python percorre itens de qualquer objeto iterável (listas, strings, tuplas, dicionários). Ele é projetado para ser legível e seguro, evitando o erro clássico de "acessar um índice fora dos limites" (Out of Bounds), muito comum em linguagens de baixo nível.

Análise de Cenários Reais e Aplicação

Em uma Sprint de desenvolvimento, usamos for para gerar relatórios mensais (percorrendo todos os dias do mês) e usamos while para processos de monitoramento de sistema (background tasks) que precisam rodar indefinidamente até um comando de interrupção manual ou erro crítico.

transacoes = [100.50, -20.00, 50.00, -150.00, 300.00]
total_positivo = 0
for valor in transacoes:
    if valor > 0:
        total_positivo += valor
print(f"Total de depósitos: R$ {total_positivo}")
Erro Comum na Repetição

Erro Comum: Tentar modificar uma lista enquanto a percorre com um for. Isso causa comportamentos imprevisíveis porque o iterador perde a referência do próximo índice. A recomendação de mercado é sempre criar uma cópia da lista ou usar uma lista auxiliar.

Ambiente de Trabalho e Tomada de Decisão

Simulação de Ambiente de Trabalho: O uso excessivo de break e continue pode tornar o código difícil de seguir (o chamado "Spaghetti Code"). No entanto, em menus de sistemas legados via terminal, o while True: combinado com um break para sair do programa é um padrão de projeto clássico e aceitável.

Tomada de Decisão Técnica: Suponha que seu sistema de Caixa Eletrônico tenha um limite de 3 tentativas de senha. Você usará um for de 1 a 3? Ou um while com um contador? A decisão impacta na legibilidade. Se o número de tentativas é fixo e conhecido, o for com range(3) é tecnicamente mais elegante e menos propenso a erros de incremento de variável.

Discussão Orientada e Conexão

Discussão Orientada: Por que o Python não tem uma estrutura do-while como outras linguagens? A filosofia do Python (The Zen of Python) preza por "Haver apenas uma maneira óbvia de fazer algo". O while True com um if/break no final supre perfeitamente a necessidade de um do-while, mantendo a linguagem enxuta.

Conexão com o Projeto: Num sistema industrial, se um sensor detectar fumaça, não podemos esperar o loop de monitoramento de temperatura terminar. Precisamos de um break no fluxo normal e um redirecionamento imediato para a rotina de emergência.

Consolidação e Desafio: Engenharia da Solução "ATM System"

Refinamento Técnico e Boas Práticas

Refinamento Técnico: Nesta última hora, vamos consolidar o conhecimento unindo tudo. Um sistema real de Caixa Eletrônico (ATM) exige um loop principal de interface, decisões de saque e validação de estado (saldo). Vamos aplicar o conceito de "acumuladores" e "sentinelas". Uma variável sentinela é aquela que, ao mudar de valor, sinaliza o fim de um processo (como a palavra "sair").

  • Nomes de Variáveis Semânticos: Use valor_saque em vez de vs.
  • Constantes: Use LIMITE_DIARIO = 500 em letras maiúsculas para valores que não mudam.
  • F-strings: Utilize a formatação moderna f"Texto {variavel}" para logs claros.
  • Refatoração: Se o seu if está ficando muito grande, talvez seja hora de quebrá-lo em funções (assunto das próximas aulas).
Preparação e Execução Orientada

Preparação para o Desafio: Vocês devem estruturar o programa para que ele seja resiliente. O que acontece se o usuário digitar uma letra em vez de um número no valor do saque? Embora ainda não tenhamos visto tratamento de exceções (try/except) a fundo, o uso de if valor.isdigit() pode ser uma solução elegante para validar strings antes da conversão.

Execução Orientada: O fluxo lógico sugerido para o desafio é:
1. Inicializar variáveis de estado (saldo inicial).
2. Abrir um laço while True.
3. Exibir o menu de opções.
4. Capturar a entrada do usuário.
5. Avaliar com if/elif a opção escolhida.
6. Realizar os cálculos e atualizações de saldo.
7. Usar break para encerrar o programa quando solicitado.

FAQ

Dúvidas Reais de Alunos

1. "Professor, por que usar while True? Isso não vai travar o computador?"
Resposta: O while True cria um loop potencialmente infinito, mas ele só "trava" se você não oferecer uma saída. Quando usamos o comando break dentro de um if, o Python encerra o laço na hora. É a forma padrão de criar menus interativos que só fecham quando o usuário decide.
2. "Qual a diferença real entre elif e vários if seguidos?"
Resposta: Excelente pergunta! O elif faz parte de um mesmo bloco lógico. Se a primeira condição for verdadeira, o Python nem olha as outras, o que economiza tempo. Vários if seguidos são independentes; o Python testará todos eles, mesmo que o primeiro já tenha resolvido o problema, o que pode causar erros lógicos se as condições se sobrepuserem.
3. "Eu posso colocar um for dentro de um if que está dentro de um while?"
Resposta: Sim, isso se chama "Aninhamento" (Nesting). É muito comum, mas cuidado: se você aninhar demais, seu código vira um "Código de Orgulho do Avô" (impossível de entender). A dica de mercado é tentar manter no máximo 2 ou 3 níveis de aninhamento.
4. "O continue apaga o que eu fiz na repetição?"
Resposta: Não! Ele apenas ignora o que vem depois dele dentro daquele ciclo específico da repetição e volta para o teste da condição no topo do laço. Tudo o que foi somado ou alterado antes da linha do continue permanece salvo na memória.

Desafio Prático

Desafio Prático: SENAIBank

Contexto da Situação: A cooperativa de crédito "SENAIBank" precisa de um protótipo funcional para seu novo terminal de autoatendimento em modo texto. O sistema deve ser amigável e impedir que o cliente retire mais dinheiro do que possui.

Objetivo Técnico: Implementar um algoritmo robusto de controle de fluxo que gerencie o estado financeiro de um usuário durante uma sessão de uso.

Requisitos Obrigatórios
  • Saldo inicial de R$ 1.000,00.
  • Menu com as opções: (1) Ver Saldo, (2) Depositar, (3) Sacar, (4) Sair.
  • O programa deve rodar continuamente até que a opção "Sair" seja escolhida.
  • No saque, validar se o valor é positivo e se há saldo disponível.
  • No depósito, validar se o valor é positivo.
  • Exibir mensagens de erro claras para operações inválidas.
Critérios de Avaliação
  • 1. Uso correto da indentação.
  • 2. Implementação funcional do laço while.
  • 3. Lógica condicional correta para validação de saldo (não permitir saldo negativo).
  • 4. Uso de break para encerramento limpo.
  • 5. Legibilidade do código (nomes de variáveis e comentários).
Passo a Passo Sugerido
1
Defina a variável saldo = 1000.0. Inicie o while True:. Imprima o menu e use input() para ler a opção. Use um bloco if/elif/else para tratar cada uma das 4 opções.
2
Dentro da opção de saque, use um if aninhado para verificar if valor_saque <= saldo:. Para a opção 4, use o comando break. Teste o programa tentando sacar R$ 1.500,00 e veja se ele nega corretamente.
