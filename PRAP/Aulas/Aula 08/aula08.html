Conceitos Fundamentais

Módulos, instrução import, __name__ e Persistência Básica.

Seção 2.1

O que são Módulos e Modularização

A modularização consiste em dividir um programa grande e complexo em partes menores, autônomas e focadas em responsabilidades específicas, que chamamos de módulos.

Em Python, qualquer arquivo com a extensão .py é considerado um módulo.

Explicação
Ao dividirmos o código dessa forma, aplicamos o conceito de separação de responsabilidades ( Separation of Concerns ).
Seção 2.1

Vantagens da Modularização

Assim, em vez de termos um arquivo gigante misturando lógicas financeiras, funções de apoio e inicialização do sistema, passamos a ter arquivos separados para cada objetivo.

Isso torna o código extremamente mais limpo, fácil de debugar e possibilita que vários programadores trabalhem no mesmo projeto simultaneamente, uma vez que não estarão todos modificando o mesmo arquivo.

Seção 2.1 · Demonstração Prática

Demonstração prática

Imagine que antes tínhamos um arquivo tudo_junto.py. Agora, criamos um arquivo chamado matematica.py apenas com as lógicas de cálculo:

Python # Arquivo: matematica.py def somar(a, b): # Retorna a soma de dois números return a + b def subtrair(a, b): # Retorna a subtração de dois números return a - b

Este arquivo, por si só, é um módulo que agora pode ser reutilizado.

Seção 2.2

A instrução import

Depois que dividimos nosso código em módulos, precisamos de uma forma para conectá-los.

A instrução import serve exatamente para carregar o código de um arquivo (módulo) para dentro de outro.

O Python nos permite importar o módulo inteiro ou apenas funções específicas daquele módulo utilizando a sintaxe from ... import .... Ao fazer essa importação, o interpretador lê o arquivo de origem e torna as suas funções, variáveis e classes disponíveis no arquivo de destino, promovendo a tão almejada reutilização de código sem precisar copiar e colar blocos de instruções.

Seção 2.2 · Demonstração Prática

Demonstração prática

Criando um arquivo principal chamado calculadora.py que utilizará o módulo matematica.py que criamos anteriormente.

Python # Arquivo: calculadora.py import matematica # Acessamos a função somar utilizando o nome do módulo seguido de um ponto resultado = matematica.somar(10, 5) print(f"A soma é: {resultado}") # Alternativa importando apenas a função específica from matematica import subtrair resultado_subtracao = subtrair(10, 5) print(f"A subtração é: {resultado_subtracao}")

Seção 2.3

A Variável Especial __name__

Quando um módulo é importado por outro, o Python executa todo o código daquele arquivo de origem.

Atenção
Se houver comandos print ou invocações de funções espalhadas pelo módulo, eles serão executados indevidamente durante a importação.

Para controlar isso, o Python possui uma variável especial oculta chamada __name__.

Seção 2.3

A Variável Especial __name__

Quando rodamos um script diretamente (ex: python app.py), a variável __name__ desse arquivo assume o valor de __main__ ( principal ).

No entanto, se o arquivo for importado por outro, a variável __name__ receberá o próprio nome do arquivo.

Solução
Assim, podemos usar um bloco condicional if __name__ == '__main__': para garantir que certos trechos de código só rodem se o arquivo for o principal.
Seção 2.3 · Demonstração Prática

Demonstração prática

Python # Arquivo: utils.py def calcular_dobro(numero): return numero * 2 # Esse bloco impede que o print seja rodado quando houver um 'import utils' em outro arquivo if __name__ == '__main__': # Este teste rápido só roda se executarmos 'python utils.py' no terminal print("Testando a função:", calcular_dobro(5))

Seção 2.4

Persistência Básica (Gerenciamento de Arquivos)

Neste ponto, nossos programas operam exclusivamente na memória RAM. Isso significa que, se desligarmos o computador ou fecharmos a aplicação, todos os dados de variáveis, listas ou dicionários são perdidos.

Solução
Para resolver esse problema, introduzimos a persistência básica de dados salvando e lendo informações em arquivos de texto no disco rígido.
Seção 2.4

Manipulando Arquivos em Python

Em Python, utilizamos a função embutida open() para criar, ler e gravar arquivos.

Combinamos essa função com a palavra-chave with, que funciona como um " gerenciador de contexto ", garantindo que o arquivo seja fechado automaticamente de forma segura assim que terminarmos a nossa operação, evitando vazamentos de memória ou arquivos corrompidos.

Seção 2.4 · Demonstração Prática

Demonstração prática

Python # Arquivo: salvar_dados.py saldo = "1500.50" # Escrevendo no arquivo (Modo 'w' - write / gravar) with open("saldo.txt", "w") as arquivo: arquivo.write(saldo) print("Saldo gravado com sucesso!") # Lendo o arquivo (Modo 'r' - read / ler) with open("saldo.txt", "r") as arquivo: conteudo = arquivo.read() print(f"Saldo recuperado: R$ {conteudo}")

Demonstração Aplicada

Vamos imaginar um cenário completo combinando todos os conceitos. Desenvolveremos um pequeno gerenciador de senhas separado em módulos.

Seção 3

Lógica em utilitarios.py

Temos o arquivo das funções lógicas (utilitarios.py):

Python # Arquivo: utilitarios.py def salvar_senha(servico, senha): linha = f"{servico}:{senha}\n" # Adiciona a senha no arquivo de persistência with open("cofre.txt", "a") as arquivo: arquivo.write(linha) def listar_senhas(): # Tenta ler as senhas, se o arquivo existir try: with open("cofre.txt", "r") as arquivo: return arquivo.read() except FileNotFoundError: return "Nenhuma senha cadastrada ainda." # O bloco a seguir permite testar localmente as funções do módulo if __name__ == "__main__": print("Testando salvamento...") salvar_senha("TesteApp", "1234") print(listar_senhas())
Seção 3

Interface principal em main.py

E temos o arquivo da interface principal (main.py):

Python # Arquivo: main.py import utilitarios def iniciar_sistema(): print("--- GERENCIADOR DE SENHAS ---") opcao = input("Digite 1 para salvar ou 2 para listar: ") if opcao == "1": servico = input("Nome do serviço: ") senha = input("Digite a senha: ") utilitarios.salvar_senha(servico, senha) print("Salvo com sucesso!") elif opcao == "2": print("--- SUAS SENHAS ---") print(utilitarios.listar_senhas()) else: print("Opção inválida.") if __name__ == "__main__": iniciar_sistema()
Seção 3

Análise do Cenário

Isolamento
Neste cenário, a lógica complexa está isolada. A tela do sistema no main.py apenas coleta dados e os passa para o módulo focado na persistência e manipulação.
Resultado
Se o main.py encerra, os dados continuam intactos no cofre.txt.

FAQ – Dúvidas

Respostas rápidas para as perguntas mais comuns.

Seção 5 · FAQ

Dúvidas (1/3)

Quando eu devo dividir meu código em outro arquivo?
Sempre que o seu arquivo principal estiver ficando extenso ou quando você perceber que tem um grupo de funções que possuem um tema em comum (como funções matemáticas, ou acesso a banco de dados).
Eu posso fazer um import de um arquivo que está em outra pasta?
Sim. No Python, podemos estruturar pastas como "pacotes" utilizando arquivos especiais, permitindo o import através do formato from pasta.arquivo import funcao.
Seção 5 · FAQ

Dúvidas (2/3)

Por que usamos with open e não apenas arquivo = open()?
Se você usar apenas o open(), precisará lembrar de escrever a instrução arquivo.close() no final. Se ocorrer um erro no meio do programa e o arquivo não fechar, ele pode ficar travado pelo sistema operacional ou corromper dados. O with é uma garantia de fechamento automático.
O que significa o modo w, r e a na manipulação de arquivos?
Eles indicam o propósito da abertura. r (read) é exclusivo para leitura. w (write) grava informações substituindo tudo que existia no arquivo. a (append) escreve a informação no final do arquivo sem apagar os dados antigos.
Seção 5 · FAQ

Dúvidas (3/3)

Se o arquivo de texto não existir, o Python cria pra mim?
Nos modos de gravação (w e a), o Python cria automaticamente o arquivo se ele não existir. Porém, no modo de leitura (r), se o arquivo não for encontrado o programa vai estourar um erro (FileNotFoundError).
