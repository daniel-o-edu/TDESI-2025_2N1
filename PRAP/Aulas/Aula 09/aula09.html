02

Conceitos Fundamentais

A base estrutural da Programação Orientada a Objetos.

Seção 2.1

O que são Classes e Objetos (O Molde e a Peça)

Uma Classe funciona como um molde, uma planta baixa ou um projeto abstrato que define as características e os comportamentos de algo que queremos representar no sistema.

Por exemplo, a planta de uma casa define onde ficam os quartos e portas, mas não é a casa física onde você pode morar.

Já o Objeto (também chamado de instância) é a casa construída a partir dessa planta; é a materialização da classe na memória do computador, possuindo dados reais e específicos.

Seção 2.1

A Abordagem Orientada a Objetos

Na Programação Orientada a Objetos, nós primeiro desenhamos as classes para estipular o que as nossas entidades (como Cliente, Produto ou Veículo) terão, e depois criamos quantos objetos precisarmos a partir desse molde único.

Benefício Lógico
Essa abordagem organiza incrivelmente o pensamento lógico, pois passamos a programar pensando em "coisas" que interagem entre si, aproximando a linguagem do computador da forma como nós, humanos, compreendemos o mundo real.

Seção 2.2

O Método Construtor (__init__) e Atributos

Adicionar variáveis soltas nos objetos, como fizemos acima, não é uma boa prática profissional porque cada objeto pode acabar com variáveis diferentes, quebrando a padronização.

Para resolver isso, utilizamos o método construtor, que em Python é escrito como __init__ ( com dois underlines de cada lado).

Este é um método especial que é executado automaticamente no exato momento em que um objeto é instanciado.

Seção 2.2

Inicializando o "Estado" do Objeto

O seu papel principal é inicializar os atributos da classe, ou seja, as variáveis internas que guardarão o "estado" do objeto.

Dentro do construtor, utilizamos a palavra obrigatória self, que é uma referência ao próprio objeto que está sendo criado naquele momento.

Entendendo a Lógica
Ao declararmos self.nome = nome, estamos dizendo: "pegue a variável 'nome' que o programador passou e guarde-a permanentemente dentro da memória DENTRO deste objeto específico".
Seção 2.2 · Demonstração Prática

Criando um Construtor Padronizado

Vamos criar uma classe com um construtor padronizado:

class Funcionario: # O método __init__ é chamado na hora da criação # O 'self' vai automaticamente, nós só precisamos passar 'nome' e 'cargo' def __init__(self, nome, cargo): self.nome = nome # self.nome é o atributo permanente do objeto self.cargo = cargo # self.cargo é o atributo permanente do objeto # Criando o objeto e já passando os dados obrigatórios do construtor func1 = Funcionario("Carlos", "Desenvolvedor Backend") func2 = Funcionario("Ana", "Gerente de Projetos") print(f"{func1.nome} atua como {func1.cargo}")

Seção 2.3

Métodos (Comportamentos do Objeto)

Se os atributos são as características (adjetivos/dados) de um objeto, os métodos são as ações (verbos/comportamentos) que este objeto consegue realizar.

Tecnicamente, um método é simplesmente uma função definida dentro de uma classe.

O Parâmetro 'self'
A grande diferença para uma função comum é que todo método de instância em Python precisa receber a palavra self como o seu primeiro parâmetro.
Seção 2.3

Encapsulando as Regras de Negócio

Graças a esse self, o método consegue enxergar e modificar os atributos internos daquele objeto específico sem precisar que os dados sejam passados repetidamente como parâmetros externos.

O Poder do Encapsulamento
É através dos métodos que encapsulamos as regras de negócio: em vez de alterar o saldo de um cliente manualmente fora da classe, nós pedimos para o objeto cliente executar a ação de "depositar", garantindo que todas as validações de segurança daquele método sejam executadas pela própria classe.
Seção 2.3 · Demonstração Prática

Adicionando um Comportamento

class ContaSimples: def __init__(self, titular, saldo_inicial): self.titular = titular self.saldo = saldo_inicial # Este é um método da classe def depositar(self, valor): if valor > 0: self.saldo += valor print(f"Depósito de R${valor} realizado com sucesso!") else: print("Valor inválido para depósito.")
Seção 2.3 · Demonstração Prática

Invocando o Comportamento

def exibir_extrato(self): print(f"Titular: {self.titular} | Saldo atual: R${self.saldo}") conta = ContaSimples("Roberto", 100) conta.depositar(50) # Chamando o método do objeto conta.exibir_extrato()

Seção 2.4

Herança (Reaproveitamento de Código)

A Herança é um dos pilares mais poderosos da Orientação a Objetos.

Ela permite criarmos uma nova classe (chamada de subclasse ou classe filha) que absorve automaticamente todos os atributos e métodos de uma classe já existente (chamada de superclasse ou classe mãe).

Ao utilizar a herança, eliminamos a repetição de código.

A Relação "É UM"
Pense nisso como uma relação categórica do tipo " É UM ": um Gerente " é um " Funcionário; uma Moto " é um " Veículo.

Seção 5

FAQ – A Palavra 'self'

O que significa exatamente a palavra self?

O self é como o objeto chama a si mesmo. Pense nele como o pronome "Eu".

Quando você escreve self.nome = "João", o objeto está dizendo: "O MEU nome é João".

É a forma de garantir que o objeto modifique seus próprios dados e não os dados de outro objeto acidentalmente.

Seção 5

FAQ – Obrigatoriedade do __init__

É obrigatório usar o __init__ em toda classe que eu criar?

Não é estritamente obrigatório (o Python aceita classes sem ele), mas no mercado de trabalho é uma regra implícita.

Um objeto sem estado inicial (sem __init__) costuma não ser muito útil e pode causar bugs silenciosos.
