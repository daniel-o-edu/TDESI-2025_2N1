Aula 12 - (CRUD) Inserção e Leitura

Descrição: Nesta aula, vamos trabalhar os comandos fundamentais do CRUD (Create e Read) executados pela linguagem Python, aplicando as instruções INSERT e SELECT em um banco de dados relacional (SQLite).

O foco principal está no aprimoramento do uso de coleções: o retorno bruto de um comando SELECT será processado, iterado e transformado em uma Lista de Dicionários para promover abstração, segurança na manutenção e fácil manipulação dos dados pelo sistema.

Tópicos desta aula

Comandos CRUD (Create e Read)
Instruções INSERT e SELECT (SQLite)
Processamento de retorno bruto
Lista de Dicionários
O sistema deverá cadastrar os itens permanentemente no banco de dados e, na funcionalidade de listagem, buscar esses registros persistidos, convertê-los em listas de dicionários e exibi-los estruturadamente utilizando um laço de repetição for.

2. Conceitos Fundamentais e Base Técnica

Conceito 1

Conceito 1: Inserção de Dados e Persistência (Comando INSERT)

A. Pergunta Desafiadora:

"Se a energia do seu computador cair neste exato segundo, como o aplicativo do banco garante que o pix que você acabou de fazer não desapareceu no meio do caminho e foi salvo com sucesso?"

B. Cenário Real:

No desenvolvimento de um módulo de estoque para uma franquia de farmácias, cada vez que o leitor de código de barras bipa um novo remédio que chegou do caminhão, um aplicativo em Python recebe essa string. Para garantir que esse remédio conste no balanço da loja no dia seguinte, o Python precisa acionar o banco de dados imediatamente através de um comando INSERT, registrando a chegada sem margem para falhas voláteis da memória.

Explicação

C. Explicação Teórica:

A transição de dados temporários para armazenamento persistente é um marco fundamental no desenvolvimento de sistemas reais.

Quando utilizamos apenas variáveis e coleções em memória RAM, como listas e dicionários básicos, qualquer interrupção de energia ou o simples fechamento do aplicativo resulta na perda irreversível de todas as informações cadastradas.

Para solucionar esse problema crítico de persistência, os softwares corporativos utilizam Sistemas de Gerenciamento de Banco de Dados (SGBDs), que gravam as informações de forma estruturada e definitiva nos discos rígidos ou em unidades de estado sólido (SSD) dos servidores.

Explicação

C. Explicação Teórica (cont.):

O comando de inserção de dados, amplamente conhecido como INSERT na linguagem SQL, é a ponte direta entre o aplicativo em Python e a tabela de destino no banco de dados.

No contexto da programação estruturada moderna, o Python utiliza uma biblioteca nativa (como a sqlite3) para abrir um canal de comunicação seguro, pelo qual os dados coletados das telas ou terminais dos usuários são enviados.

O gerenciador de conexão estabelecido cria um objeto especial chamado "cursor", que atua como um mensageiro virtual encarregado de transportar e executar a instrução SQL exata juntamente com seus parâmetros até o núcleo do banco de dados.

Explicação

C. Explicação Teórica (cont.):

Contudo, não basta apenas enviar o comando para o banco de dados; é imperativo confirmar essa transação de forma explícita na arquitetura do software.

O conceito de commit, ou confirmação de transação, garante que o SGBD realmente efetive a gravação física dos dados nas tabelas, prevenindo corrompimentos caso ocorra uma falha grave durante o processo elétrico ou lógico.

Sem a invocação metódica do comando commit() após a instrução de INSERT no Python, as modificações ficam isoladas em uma memória temporária transitória do banco e são sumariamente e automaticamente descartadas assim que a conexão principal for finalizada pelo desenvolvedor.

Técnica

D. Explicação Técnica:

No Python, utilizamos a biblioteca sqlite3. O processo segue um ciclo de vida restrito:

  • connect() para abrir o arquivo de banco;
  • cursor() para criar o executor;
  • execute() para disparar uma query parametrizada (INSERT INTO tabela (campos) VALUES (?, ?)) injetando uma Tupla para mitigar ataques de SQL Injection;
  • commit() para confirmar; e
  • close() para liberar o arquivo do Sistema Operacional.

A interrogação (?) é a melhor prática de segurança e atua como uma âncora posicional.

Código

E. Exemplo de Código:

Python import sqlite3 #1. Estabelecendo a conexão conexao = sqlite3.connect("meu_sistema.db") cursor = conexao.cursor() #2. Dados hipotéticos nome_produto = "Teclado Mecânico" preco_produto = 250.00 #3. Garante que a tabela exista cursor.execute(""" CREATE TABLE IF NOT EXISTS produtos ( id INTEGER PRIMARY KEY AUTOINCREMENT, nome TEXT NOT NULL, preco REAL NOT NULL ) """) #4. Execução da query com passagem segura comando_sql = "INSERT INTO produtos (nome, preco) VALUES (?, ?)" valores = (nome_produto, preco_produto) cursor.execute(comando_sql, valores) #5. Confirmação e Encerramento conexao.commit() cursor.close() conexao.close() print("Produto persistido com sucesso!")

Conceito 2

Conceito 2: Leitura de Dados e Transformação em Dicionários (SELECT e fetchall)

A. Pergunta Desafiadora:

"Se você pedisse um prato em um restaurante e o garçom te trouxesse os ingredientes todos misturados em uma bacia genérica e sem rótulos, como você identificaria o que comer? Como transformar dados brutos em algo organizado que nosso sistema consegue ler pelo nome?"

B. Cenário Real:

Ao abrir um aplicativo de Delivery, a primeira tela exibe a lista de restaurantes com suas respectivas notas e taxas de entrega. O Python do servidor vai ao banco, faz um SELECT e recebe centenas de informações embaralhadas em matrizes (Tuplas de Tuplas). Antes de mandar para o aplicativo no celular (Frontend), o desenvolvedor backend itera essas matrizes, transformando-as em uma Lista de Dicionários bem desenhada (JSON) onde o celular consiga ler facilmente restaurante["nome"] e restaurante["taxa_entrega"].

Explicação

C. Explicação Teórica:

Após estabelecermos a capacidade de gravar dados de forma persistente em um banco, o próximo desafio lógico é a capacidade de recuperar essas informações com precisão, integridade e velocidade.

A operação de leitura, padronizada pelo comando estruturado SELECT na linguagem SQL, permite que o sistema Python interrogue o banco de dados local ou remoto e traga em milissegundos conjuntos de registros que atendam a critérios estritamente definidos.

Quando utilizamos funções nativas do cursor como o fetchall(), o banco de dados finaliza a requisição e devolve as informações organizadas tecnicamente em um formato de tuplas dentro de uma grande lista, as quais representam linhas e colunas brutas extraídas da tabela do SGBD.

Explicação

C. Explicação Teórica (cont.):

Embora as tuplas sejam estruturas de dados incrivelmente eficientes em termos de processamento algorítmico e baixo consumo de memória interna, elas definitivamente não são a organização mais amigável ou segura para a manipulação de regras de negócios em um aplicativo Python moderno.

O acesso aos dados em uma tupla precisa ser feito unicamente por meio de índices numéricos invisíveis (como linha [0] para ID e linha [1] para o Nome), o que torna o código-fonte altamente opaco e extremamente complexo para a compreensão humana.

Se a ordem das colunas no banco de dados sofrer alterações por alguma atualização na infraestrutura, todo o código do sistema baseado em índices numéricos inevitavelmente quebrará em tempo de execução, gerando falhas operacionais.

Explicação

C. Explicação Teórica (cont.):

A solução madura, canônica e implementada por desenvolvedores seniores no cenário global da tecnologia da informação para este tipo de risco é a conversão explícita dos dados brutos do formato tupla para uma "Lista de Dicionários".

Ao percorrermos o resultado cru do banco de dados utilizando um laço de repetição condicional como o for, conseguimos mapear de forma segura chaves textuais e descritivas para cada índice recebido (por exemplo, associando "nome_produto": linha [1]).

O dicionário resultante fornece uma camada de significado profunda e inquebrável, permitindo que qualquer outra função ou interface de usuário web acesse a informação utilizando a string com o nome semântico do dado, blindando a arquitetura do software contra falhas por desorganização visual ou mecânica.

Técnica

D. Explicação Técnica:

Quando chamamos cursor.execute("SELECT * FROM produtos") seguido de dados_brutos = cursor.fetchall(), o retorno de dados_brutos se parecerá com: [(1, 'Mouse', 50.0), (2, 'Teclado', 150.0)].

Ao instanciar uma lista vazia e fazer um for linha in dados_brutos:, construímos dicionários temporários onde nomeamos os índices [0], [1], [2] com strings que equivalem aos campos.

Utilizamos o método append() para povoar a nova lista tratada, garantindo fácil acesso via dicionario["chave"].

Código

E. Exemplo de Código:

Python import sqlite3 conexao = sqlite3.connect("meu_sistema.db") cursor = conexao.cursor() #1. Busca todos os registros cursor.execute("SELECT id, nome, preco FROM produtos") # Ajustado o nome do campo id resultado_bruto = cursor.fetchall() #2. Processo de Transformação lista_produtos_tratada = [] for linha in resultado_bruto: produto_dicionario = { "id": linha[0], "nome": linha[1], "preco_unitario": linha[2] } lista_produtos_tratada.append(produto_dicionario) conexao.close() #3. Consumo dos dados print("--- RELATÓRIO DO SISTEMA ---") for produto in lista_produtos_tratada: print(f"[{produto['id']}] Item: {produto['nome']} | Valor: R$ {produto['preco_unitario']:.2f}")

3. FAQ - Dúvidas Reais de Programação e Banco de Dados

FAQ

Dúvidas Reais (Parte 1)

  • Pergunta 1: Eu sempre preciso usar a interrogação (?) no INSERT? Por que não usar F-Strings direto?
    Resposta: Nunca utilize f-strings em queries SQL. O uso direto de variáveis em f-strings na SQL abre o software para a maior brecha de segurança cibernética: O SQL Injection. Hackers podem inserir comandos destrutivos (como DROP TABLE) disfarçados de nomes de usuários na tela, destruindo seu banco. O ? força a biblioteca do banco a sanitizar a variável com segurança extrema.
  • Pergunta 2: O que exatamente o comando commit faz nos bastidores da Máquina?
    Resposta: Bancos de dados profissionais adotam propriedades chamadas ACID (Atomicidade, Consistência, Isolamento e Durabilidade). O commit é a garantia de Durabilidade. Até invocar o commit(), seus dados ficam trafegando nas memórias voláteis do gerenciador do banco. Somente o commit comanda a gravação magnética real no disco de estado sólido, blindando a modificação de falhas elétricas abruptas.
FAQ

Dúvidas Reais (Parte 2)

  • Pergunta 3: Qual a diferença entre parâmetro e argumento ao executar uma query?
    Resposta: O parâmetro é o "lugar vazio" esperado, ou seja, o ponto de interrogação ? no comando INSERT INTO valores (?, ?). O argumento é o dado concreto (Tupla("Maria", 30)) que você fornece para o cursor.execute(). O argumento preenche materialmente o vazio reservado pelo parâmetro na query.
  • Pergunta 4: O que acontece se eu esquecer de chamar o método close() após terminar o SELECT?
    Resposta: O arquivo.db continuará ocupado e trancado (lock de arquivo) pelo processo do Python perante o Sistema Operacional. Isso gerará lentidão, retenção de memória desnecessária (Memory Leak) no servidor e possivelmente impedirá que outras partes do sistema alterem os arquivos gerando o famoso erro de concorrência "database is locked".
FAQ

Dúvidas Reais (Parte 3)

  • Pergunta 5: Por que convertemos tuplas para Dicionários se o código fica maior? Isso não é ineficiente?
    Resposta: Em desenvolvimento profissional, tempo de leitura (humanos lendo o código) é imensamente mais custoso que microssegundos de máquina. Transformar tuplas em dicionários melhora brutalmente a "Legibilidade do Código", respeitando a filosofia de Clean Code. Manutenção baseada em rótulos ({"idade":25}) é segura e auditável, basear lógica corporativa em índices (linha [14]) leva inevitavelmente a quebras críticas em futuras atualizações arquiteturais.
